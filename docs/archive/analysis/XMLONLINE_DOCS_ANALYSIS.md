## FAQ - Часто задаваемые вопросы {#faq}

### Пояснительная записка. Версия 01.03 (НЕАКТУАЛЬНАЯ)

#### 1. Ошибки в номере Приказа Минстроя РФ об утверждении классификатора

**Вопрос:** Пояснительная записка выгружается с ошибками, в т.ч. в номере Приказа Минстроя РФ об утверждении классификатора объектов капитального строительства

**Ответ:**
- За визуальное отображение Пояснительной записки в формате XML отвечает схема XSL, расположенная на сайте Минстроя: https://www.minstroyrf.gov.ru/docs/183387/
- В данной схеме всегда выводится фраза про классификатор "... от 10.07.2020 №374/пр"
- Если изменить данный блок, будут разночтения между разными сервисами
- В самом XML-файле не указано, какой именно классификатор используется
- В XML-файле указан только код функционального назначения
- Все фразы про даты и № приказа содержатся только в XSL-схеме
- **Это не является ошибкой, ничего править не нужно**

#### 2. Обязательность ТЭП (технико-экономических показателей)

**Вопрос:** Действительно ли технико-экономические показатели необязательны для заполнения?

**Ответ:**
- Техническими возможностями сервиса предусмотрено не заполнять поля с ТЭП, если технико-экономические показатели по объекту не требуются
- **НО:** для всех строящихся и реконструируемых объектов, для которых требуется получение Разрешения на строительство, заполнение ТЭП **ОБЯЗАТЕЛЬНО**

#### 3. Требования по предоставлению ПЗ в формате XML

**Вопрос:** Почему при подаче документов для прохождения экспертизы не было выставлено требование по предоставлению ПЗ в формате XML?

**Ответ:**
- С 01.08.2023г представление на экспертизу первого раздела проектной документации "Пояснительная записка" требуется в формате XML
- Если задание на проектирование было утверждено до 27.05.2023г, действовал переходный период
- С 27.01.2024 г введена в действие новая схема заключения, при которой регистрация без Пояснительной записки в формате XML невозможна, если задание на проектирование утверждено после 01.09.2022г

#### 4. Некапитальные объекты

**Вопрос:** Если проектной документацией предусмотрено проектирование некапитального объекта?

**Ответ:**
- Обязательной экспертизе подлежит только проектная документация для объектов капитального строительства
- Регистрация положительного заключения экспертизы в ЕГРЗ для некапитальных объектов невозможна
- Возможна выдача экспертного заключения без регистрации

#### 5. Идентификатор объекта

**Вопрос:** Что указывать в поле: Идентификатор объекта (части объекта)?

**Ответ:**
- Идентификатор объекта - это любое значение, начинающееся с символа
- Пример: `id1` (правильно)
- **НЕЛЬЗЯ** начинать с цифры: `1id` (неправильно - даст ошибку)
- Сам идентификатор нигде не выведется в ПЗ, это больше для технических обозначений

#### 6. Обязательность формирования XML на сервисе xmlonline.ru

**Вопрос:** Обязательно ли формировать XML файл на Вашем сервисе? Является ли формирование ПЗ на сервисе ССЭ автоматической подачей документов для прохождения экспертизы?

**Ответ:**
- Данный инструмент является одним из возможных, и рекомендуемых
- Формирование XML файла возможно с помощью любого удобного для вас сервиса
- После завершения необходимо **скачать файл в формате XML** и отправить его в составе подаваемых на экспертизу документов
- Доступ в личные кабинеты пользователей недоступен для экспертов
- **Формирование ПЗ НЕ является автоматической подачей документов**

## Инструкции по заполнению данных {#instructions}

### Элементы заполнения данных

При работе с XML-файлами можно выделить следующие типы полей для заполнения:

#### 1. Простые текстовые поля

- Заполняются произвольными значениями
- Некоторые поля имеют **маску ввода** для цифровых значений или дат
- Существуют **ограничения на количество символов** для некоторых параметров

![Пример простых текстовых полей](https://xmlonline.ru/storage/posts/May2024/014.png)

#### 2. Увеличенные текстовые поля

- Аналогично простым текстовым полям
- Допускается **большее количество символов** для ввода

![Пример увеличенных текстовых полей](https://xmlonline.ru/storage/posts/May2024/022.png)

#### 3. Расширенные поля

**Назначение:**
- Ввод абзацев текста
- Заголовков
- Таблиц
- Изображений

**Форматы изображений:**
- JPG
- PNG
- BMP
- GIF

**Ограничения таблиц:**
- Таблицы **не могут содержать объединенные ячейки**
- **Не допускают использование изображений** внутри ячеек

**Смена типа элементов:**
- Нажать на маленькую зеленую кнопку с иконкой "+"
- Выбрать из списка доступных элементов:
  - Абзац текста
  - Заголовок
  - Таблица
  - Изображение

![Расширенные поля](https://xmlonline.ru/storage/posts/May2024/032.png)

#### 4. Повторяющиеся поля

**Использование:**
- Некоторые параметры могут указываться более одного раза (например, ТЭПы)

**Операции:**

1. **Добавление:** Кнопка "Добавить"
2. **Удаление:** Красная кнопка с иконкой мусорного бака
   - Первый элемент удалить нельзя
   - Первый элемент можно только очистить (желтая кнопка с иконкой ластика)
3. **Сортировка/Перемещение:**
   - Зажать кнопку мыши на кнопке с иконкой "↨"
   - Переместить элемент на нужную позицию

![Повторяющиеся поля](https://xmlonline.ru/storage/posts/May2024/08.png)

#### 5. Поля выбора

**Назначение:**
- Некоторые параметры не могут находиться одновременно в одном XML документе
- Необходимо выбрать, какой параметр заполнять

**Пример:**
- Проектировщик - юридическое лицо
- Проектировщик - индивидуальный предприниматель

**Действие:**
- Нажать на соответствующий пункт
- Ниже появится возможность заполнить составные параметры

![Поля выбора](https://xmlonline.ru/storage/posts/May2024/09.png)

#### 6. Файлы

**Добавление файлов:**
1. Нажать кнопку "Добавить файл"
2. Выбрать необходимые файлы на устройстве
3. Для добавления электронных подписей:
   - Использовать кнопку "Добавить эл. подписи"
   - Кнопка находится в элементе добавленного файла

**Удаление файлов:**
- Нажать кнопку с иконкой мусорного бака

**Важно:**
- При внесении данных **информация сохраняется моментально**
- **Не нужно нажимать кнопку сохранить**
- Процесс формирования ПЗ можно отслеживать: Кнопка "Скачать → Посмотреть"

![Работа с файлами](https://xmlonline.ru/storage/posts/May2024/10.png)

- Четкая индикация версии в UI

**Приоритет:** КРИТИЧЕСКИЙ

### 2. Типы полей и валидация - основа UX

**Обязательные типы полей (6 категорий):**

1. **Простые текстовые поля:**
   - Маски ввода для дат и цифр
   - Ограничение по количеству символов
   - Валидация в реальном времени

2. **Увеличенные текстовые поля:**
   - Большее количество символов
   - Textarea вместо input

3. **Расширенные поля (Rich Text):**
   - Абзацы текста
   - Заголовки
   - Таблицы (без объединения ячеек, без изображений в ячейках)
   - Изображения (jpg, png, bmp, gif)
   - Переключатель типа элемента (кнопка "+")

4. **Повторяющиеся поля:**
   - Кнопка "Добавить"
   - Кнопка "Удалить" (красная, с иконкой корзины)
   - Кнопка "Очистить" для первого элемента (желтая, с иконкой ластика)
   - Drag-and-drop для сортировки (кнопка "↨")
   - Первый элемент нельзя удалить

5. **Поля выбора (Radio/Select):**
   - Взаимоисключающие параметры
   - Динамическое отображение связанных полей
   - Пример: юр. лицо vs ИП

6. **Файлы:**
   - Множественная загрузка
   - Добавление электронных подписей
   - Удаление файлов
   - Отображение списка загруженных файлов

**Приоритет:** ВЫСОКИЙ

### 3. Автосохранение и режимы работы

**Ключевые особенности:**
- Информация **сохраняется моментально** при вводе
- **Нет кнопки "Сохранить"** - автосохранение каждые 30 секунд
- Возможность **предпросмотра** в реальном времени ("Скачать → Посмотреть")
- **Режим хранения файлов** - возможность скачать все файлы архивом

**Требования для проекта:**
- Реализовать автосохранение (интервал 30 сек, как в CLAUDE.md)
- Индикация статуса сохранения
- Кнопка ручного сохранения (опционально, но рекомендуется)
- Предпросмотр XML в человекочитаемом формате

**Приоритет:** ВЫСОКИЙ

### 4. Валидация и бизнес-правила

**Из FAQ извлечены важные правила:**

1. **Идентификатор объекта:**
   - Должен начинаться с буквы (не цифры!)
   - Пример правильный: `id1`
   - Пример неправильный: `1id` (даст ошибку)

2. **ТЭП (технико-экономические показатели):**
   - Технически необязательны для заполнения
   - **НО обязательны** для строящихся/реконструируемых объектов с Разрешением на строительство

3. **Некапитальные объекты:**
   - Не подлежат регистрации в ЕГРЗ
   - Возможна выдача заключения без регистрации

4. **XSL-схема визуализации:**
   - Берется с сайта Минстроя
   - Не модифицируется локально
   - Фраза про классификатор "от 10.07.2020 №374/пр" - это нормально

**Требования для проекта:**
- Валидация идентификаторов (начало с буквы)
- Условная обязательность ТЭП
- Предупреждения для некапитальных объектов
- Использование официальной XSL-схемы

**Приоритет:** СРЕДНИЙ
